11. Como extrair a “arquitetura semântica” de um site premium

Isso aqui é ouro.

Arquitetura semântica não é o CSS bonito.
É o raciocínio do site.

O senhor vai olhar cada referência e responder:

1. Qual é a promessa principal da tela?
2. Qual é a ordem do raciocínio visual?
3. Qual ação a tela quer provocar?
4. Onde ela reduz ansiedade?
5. Onde ela reforça confiança?
6. Como ela distribui informação em camadas?
7. Qual parte é decoração e qual parte é funcional?

Exemplo aplicado ao seu caso:

Tela de anamnese premium

* promessa: “você vai preencher com facilidade”
* redução de ansiedade: barra de progresso + texto curto + etapas pequenas
* aumento de confiança: linguagem clara + suporte visual + aparência limpa
* funcionalidade: campos bem agrupados
* camada extra: vídeo de ajuda, banner contextual, resumo parcial

Quando o senhor faz isso em 5 referências, já consegue criar uma tela melhor do que qualquer uma delas isoladamente.

13. Como usar DevTools sem ser programador

Seu objetivo não é programar manualmente; é investigar.

Abra um site de referência e faça:

1. botão direito → Inspecionar
2. veja o bloco HTML da seção
3. veja classes e estrutura
4. veja CSS aplicado
5. vá em Network para identificar imagens, fontes, vídeos e chamadas
6. tire prints e anote o padrão
7. descreva em português o que a seção faz

Chrome DevTools e MDN deixam claro que essas ferramentas servem exatamente para inspecionar elementos, estilos e recursos carregados.  

O senhor não precisa entender tudo.
Precisa aprender a perguntar:

* esse bloco é um card ou um grid?
* esse menu é drawer ou modal?
* esse banner é sticky ou fixed?
* essa ajuda é tooltip ou side panel?
* isso é tabela, accordion ou tabs?
